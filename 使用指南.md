# Gemini CLI 多 AI 供应商使用指南

## 📋 概述

Gemini CLI 现已支持多个 AI 供应商，除了 Google 的 Gemini 模型外，还支持：
- **硅基流动 (SiliconFlow)** - 精选高质量模型
- **智谱轻言 (Zhipu AI)** - GLM 系列中文优化模型  
- **OpenAI 兼容 API** - Claude AI 等兼容服务

## 🚀 快速开始

### 1. 硅基流动 (SiliconFlow)

#### 🔑 获取 API 密钥
1. 访问 [硅基流动官网](https://siliconflow.cn/)
2. 注册账号并登录
3. 进入控制台，找到 API 密钥管理
4. 创建新的 API 密钥
5. 复制密钥备用

#### ⚙️ 配置方法

**方法一：环境变量**
```bash
export GEMINI_SILICONFLOW_API_KEY="你的API密钥"
```

**方法二：.env 文件**
```bash
# 在项目根目录创建 .env 文件
GEMINI_SILICONFLOW_API_KEY=你的API密钥
```

**方法三：配置文件**
编辑 `~/.gemini/settings.json`：
```json
{
  "selectedAuthType": "siliconflow-api-key",
  "providers": {
    "siliconflow": {
      "apiKey": "你的API密钥",
      "model": "zai-org/GLM-4.5"
    }
  }
}
```

#### 🤖 支持的模型

| 模型名称 | 特点 | 上下文长度 |
|---------|------|-----------|
| `zai-org/GLM-4.5` (默认) | 智谱轻言旗舰模型，强大中文理解 | 128K |
| `deepseek-ai/DeepSeek-R1` | DeepSeek 推理模型，复杂推理专家 | 64K |
| `Qwen/Qwen3-Coder-480B-A35B-Instruct` | 通义千问代码模型，编程专家 | 32K |
| `moonshotai/Kimi-K2-Instruct` | 月之暗面对话模型，长上下文 | 200K |

#### 💻 使用示例

```bash
# 使用默认模型（GLM-4.5）
GEMINI_SILICONFLOW_API_KEY="你的密钥" gemini "解释量子计算的基本原理"

# 使用 DeepSeek-R1 进行复杂推理
GEMINI_SILICONFLOW_API_KEY="你的密钥" gemini --model "deepseek-ai/DeepSeek-R1" "分析这个算法的时间复杂度"

# 使用 Qwen3-Coder 进行代码生成
GEMINI_SILICONFLOW_API_KEY="你的密钥" gemini --model "Qwen/Qwen3-Coder-480B-A35B-Instruct" "写一个 Python 快速排序函数"

# 使用 Kimi-K2 进行长文档分析
GEMINI_SILICONFLOW_API_KEY="你的密钥" gemini --model "moonshotai/Kimi-K2-Instruct" "总结这份长文档的要点"
```

### 3. OpenAI 兼容 API (Claude AI)

1. 选择支持 OpenAI 兼容格式的服务商（如 Claude AI 代理服务）

```bash
# 带重试的脚本
retry_gemini() {
    local max_attempts=3
    local attempt=1

    while [ $attempt -le $max_attempts ]; do
        if gemini "$@"; then
            return 0
        else
            echo "Attempt $attempt failed, retrying..."
            ((attempt++))
            sleep 2
        fi
    done

    echo "All attempts failed"
    return 1
}

# 使用重试机制
retry_gemini --model "glm-4.5" "你的问题"
```

```bash
    for file in *.txt; do
        if [ $job_count -ge $max_jobs ]; then
            wait # 等待一个任务完成
            ((job_count--))
        fi

        (
            echo "Processing $file..."
            gemini --model "glm-4.5-air" "总结: $(cat $file)" > "${file%.txt}_summary.txt"
        ) &

        ((job_count++))
    done

    wait # 等待所有任务完成
}
```

### 2. 缓存机制

```bash
# 简单的响应缓存
cache_dir="$HOME/.gemini/cache"
mkdir -p "$cache_dir"

cached_gemini() {
    local cache_key=$(echo "$*" | md5sum | cut -d' ' -f1)
    local cache_file="$cache_dir/$cache_key"

    if [ -f "$cache_file" ] && [ $(($(date +%s) - $(stat -c %Y "$cache_file"))) -lt 3600 ]; then
        echo "Using cached response..."
        cat "$cache_file"
    else
        echo "Generating new response..."
        gemini "$@" | tee "$cache_file"
    fi
}
```

### 3. 流式处理

```bash
# 处理大文件时使用流式输出
process_large_file() {
    local file="$1"
    local chunk_size=1000

    split -l $chunk_size "$file" chunk_

    for chunk in chunk_*; do
        echo "Processing $chunk..."
        gemini --model "moonshotai/Kimi-K2-Instruct" "分析这部分内容: $(cat $chunk)"
        rm "$chunk"
    done
}
```
